关键字

const

  • const T* 指向常量的指
  • T* const 常量指针
  • const T& 常量引用,避免拷贝与修改
  • 修饰成员函数,使其无法修改成员变量

static

  • 静态成员变量:存储在全局数据区,类的所有对象所共有,只能在类体外初始化
  • 静态成员函数:不具有this指针,仅可访问静态成员变量与静态成员函数
  • 静态全局变量:存储在全局数据区,文件内可见的,文件外不可见
  • 静态局部变量:存储在全局数据区,定义它的函数或语句块结束时,其作用域结束
  • 静态函数:文件内可见的,文件外不可见

inline

  • 消除函数调用
  • 不需要寄存器来传递参数

volatile

  • 不允许被优化,与内存进行交互,防止不可控因素导致变化

explicit

  • 防止隐式转换,复制初始化

面向对象

多态

  • 覆盖
    • 虚函数:允许基类指针调用子类函数
      • 纯虚函数:必须实现的接口
      • 虚函数表:生成类对象自动的将类对象前4/8个字节(vptr)设置为虚函数表的地址,类的所有对象共享虚函数表,保存在.data段
  • 重载(同名函数):编译时将相同函数名按照一定的格式改变成可以区分

内存分配

brk,sbrk

  • program break进程堆末尾地址,sbrk函数移动program break
  • 是否立即归还给系统受M_TRIM_THRESHOLD,高地址端内存释放

mmap

  • 页对齐
  • munmap后会立即返回给系统

类型转换

  • static_cast:良性转换,
  • dynamic_cast:基类对象指针（或引用）cast到继承类指针,下行安全
  • const_cast:去掉表达式const或volatile
  • reinterpret_cast:二进制位重新解释

锁

  • 互斥锁
  • 条件锁:满足信号量,唤醒一个因为该条件而被阻塞的线程
  • 自旋锁
  • 读写锁
  • 递归锁

杂项

对齐

  • struct:每个数据成员存储的起始位置是自身大小的整数倍,结构体总大小是该结构体成员中最大的对齐模数的整数倍
  • union:最大长度的成员与类型长度最大的成员共同决定

设计模式

  • 单一职责原则:有且仅有一个原因引起类的变更
  • 里氏替换原则:子类完美替换基类
  • 依赖倒置原则:高层模块不应该依赖底层模块，二者都该依赖其抽象；抽象不应该依赖细节；细节应该依赖抽象
  • 接口隔离原则:类间的依赖关系应该建立在最小的接口上
  • 迪米特法则:类间解耦
  • 开放封闭原则:尽量通过扩展软件实体来解决需求变化
