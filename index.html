database

MYSQL

innodb引擎

  • 提供事物ACID

    • A原子性:undo log
    • C一致性:保证状态转移有效
    • I隔离性:锁机制,mvcc
    • D持久性:redo log

  • 隔离机制

    级别 脏读 不可重复读 幻读
    读未提交 o o o
    读提交 x o o
    可重复读 x x o
    序列化 x x x

    数据读取问题

    • 脏读:读取未提交事务修改过的数据
    • 不可重复读:前后多次读取数据内容不一致
    • 幻读:前后多次读取数据总量不一致

    隔离级别

    • 读未提交:读取其他未提交事务的修改
    • 读提交:只能读取其他已提交事务的修改
    • 可重复读:事务不会读到其他事务对已有数据的修改
    • 序列化:事务顺序执行

    mvcc

    • 增加列:事务ID(递增),回滚段指针
    • 读提交实现:每个语句开始将所有活跃事物读入ReadView
    • 可重复读实现:每个事物开始将所有活跃事物读入ReadView
    • 记录最早最晚活跃事物ID并判断记录ID是否在范围内且在ReadView内

MyISAM

区别

  • 不支持事务,外键,聚集索引,行级锁
  • 保存具体行数

优化

  • 字段:避免NULL,使用枚举或整数代替字符串
  • 索引:不用外键,最好不用字符字段
  • 查询:慢查询日志分析,LIMIT分页
  • 读写分离:主增改删,从查

Redis

数据类型

  • string
    • 获取长度复杂度O(1)
    • 支持扩容
    • 空间预分配
    • 惰性空间释放
  • list
    • 获取长度复杂度O(1)
    • 双端链表
    • 多态(复制,释放,对比判断)
  • hash
    • 字典底层实现
    • 链地址法
    • 重新散列进行扩展和收缩
  • skiplist
    • 随机层级索引

持久化

  • RDB(Redis DataBase):默认,按照一定的时间将内存的数据以快照的形式保存到硬盘
  • AOF(Append Only File):将Redis执行的每次写命令记录到单独的日志文件

事物

  • 一次性，顺序性，排他性的执行一个队列中的一系列命令
  • 不支持回滚

mysql and redis

  • mysql:关系型数据库,持久化数据,硬盘存储
  • redis:非关系型数据库,缓存(内存)存储

关键字

const

  • const T* 指向常量的指
  • T* const 常量指针
  • const T& 常量引用,避免拷贝与修改
  • 修饰成员函数,使其无法修改成员变量

static

  • 静态成员变量:存储在全局数据区,类的所有对象所共有,只能在类体外初始化
  • 静态成员函数:不具有this指针,仅可访问静态成员变量与静态成员函数
  • 静态全局变量:存储在全局数据区,文件内可见的,文件外不可见
  • 静态局部变量:存储在全局数据区,定义它的函数或语句块结束时,其作用域结束
  • 静态函数:文件内可见的,文件外不可见

inline

  • 消除函数调用
  • 不需要寄存器来传递参数

volatile

  • 不允许被优化,与内存进行交互,防止不可控因素导致变化

explicit

  • 防止隐式转换,复制初始化

面向对象

多态

  • 覆盖
    • 虚函数:允许基类指针调用子类函数
      • 纯虚函数:必须实现的接口
      • 虚函数表:生成类对象自动的将类对象前4/8个字节(vptr)设置为虚函数表的地址,类的所有对象共享虚函数表,保存在.data段
  • 重载(同名函数):编译时将相同函数名按照一定的格式改变成可以区分

内存分配

brk,sbrk

  • program break进程堆末尾地址,sbrk函数移动program break
  • 是否立即归还给系统受M_TRIM_THRESHOLD,高地址端内存释放

mmap

  • 页对齐
  • munmap后会立即返回给系统

类型转换

  • static_cast:良性转换,
  • dynamic_cast:基类对象指针（或引用）cast到继承类指针,下行安全
  • const_cast:去掉表达式const或volatile
  • reinterpret_cast:二进制位重新解释

锁

  • 互斥锁
  • 条件锁:满足信号量,唤醒一个因为该条件而被阻塞的线程
  • 自旋锁
  • 读写锁
  • 递归锁

对齐

  • struct:每个数据成员存储的起始位置是自身大小的整数倍,结构体总大小是该结构体成员中最大的对齐模数的整数倍
  • union:最大长度的成员与类型长度最大的成员共同决定

设计模式

  • 单一职责原则:有且仅有一个原因引起类的变更
  • 里氏替换原则:子类完美替换基类
  • 依赖倒置原则:高层模块不应该依赖底层模块，二者都该依赖其抽象；抽象不应该依赖细节；细节应该依赖抽象
  • 接口隔离原则:类间的依赖关系应该建立在最小的接口上
  • 迪米特法则:类间解耦
  • 开放封闭原则:尽量通过扩展软件实体来解决需求变化

内存管理

进程线程

区别

  • 进程是资源分配的最小单位,线程是CPU调度的最小单位
  • 进程间不会相互影响，一个线程挂掉将导致整个进程挂掉
  • 不同进程间数据很难共享,同一进程下不同线程间数据很易共享

进程通信

  • 匿名管道:半双工,单向流动,父子关系进程,字节流,大小限制
  • 有名管道:允许无亲缘关系进程
  • 消息队列:内核中消息队列的标识,链表,有格式的数据,大小限制,独立于发送和接收进程
  • 信号量:计数器,控制多个进程对共享资源的访问,不同线程之间的同步
  • 共享内存:映射一段能被其他进程所访问的内存,配合信号量
  • 套接字:不同机器间的进程通信
  • 信号:可以在任何时候发送给某一进程,由内核保存

I/O多路复用

  • select
    • O(n)无差别轮询链表
    • 用户和内核空间传递数据结构
  • epoll
    • 事件驱动:callback函数
    • 内核和用户空间共享内存
    • 红黑树维护socket
    • 链表存储就绪事件
    • epoll_create创建一个epoll实例并返回相应的文件描述符
    • epoll_ctl控制某个epoll文件描述符上的事件
    • epoll_wait等待事件触发
    • 水平触发:缓冲内容非空读就绪,缓冲区还非满写就绪
    • 边缘触发:缓冲区空转非空读通知,缓冲区满转非满写通知

切换特点

  • 不同:进程切换涉及虚拟地址空间(需切换页目录)
  • 相同:都需要切换上下文(寄存器,计数器,用户栈|内核栈,内核数据结构)

Linux命令

top

  • pid: 进程id
  • user: 所有者
  • pr: 进程优先级
  • ni: nice值,负高正低
  • virt: 虚拟内存总使用量(kb,swap+res)
  • res: 未换出的物理内存大小(kb,code+data)
  • shr: 共享内存大小
  • 状态: D不可中断的睡眠状态 R运行 S睡眠 T跟踪/停止 Z僵尸进程
  • %cpu: CPU时间占比
  • %mem: 物理内存使用占比
  • time+: 使用cpu总时间(0.01s)
  • command: 进程名称命令

SYN泛洪攻击

  • 服务器在收到SYN后分配资源,收不到ACK保持半开通连线
  • 降低SYN timeout
  • 设置SYN cookie
  • 限制一段时间内同一IP的请求数量

close_wait

  • 被动关闭的一方收到FIN包后回复ACK,进入CLOSE_WAIT
  • 关闭对应socket

time_wait

  • 主动关闭的一方收到FIN包后回复ACK,进入TIME_WAIT
  • 等待2MSL时间后进入CLOSED
  • 防止前一个连接延迟或者丢失重传的数据包被后面的连接复用
  • 确保连接方能在时间范围内关闭自己的连接

keepalive

  • 启动一个计时器,到达0之后发出TCP探测包
  • 纯ACK包Seq号与上一个包重复
  • 探测连接的对端是否存活
  • 防止中间设备因超时删除连接相关的连接表

可靠性

  • 超时重传:RTT
  • 最大消息长度:约定发送单位的最大长度
  • 滑动窗口:包乱序

拥塞控制

  • 慢启动:窗口大小随RTT指数级增长
  • 慢启动门限:窗口大小加法增加
  • 网络拥塞:重传定时器超时,重新回到慢启动
  • 快速重传:连续收到3个或以上的dup ACK判断报文丢失
  • 快速恢复:恢复到门限
